# Skill management uses existing Codex Helper state

Skill management metadata will live in the existing Codex Helper state alongside provider and Agent Client configuration, while Shared Skill content lives under Codex Helper's fixed Skill Library Root, such as `~/.codex/config-manager/skills`. Shared Skill directories in the Skill Library use a safe slug derived from the Skill Identity, while the Skill metadata remains the source for the actual identity; existing Agent Client exposure paths keep their original directory names when a Client Skill is promoted. This keeps Agent Client configuration, Skill Locations, Sharing Scope, Exposure Registry, Origin, and health state in one management model, while keeping canonical Skill files under Codex Helper ownership rather than inside any Agent Client's native skill directory.
